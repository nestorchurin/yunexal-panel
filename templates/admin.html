Cloudflare Under Attack Mode

1. Find your Zone ID

Log in to the Cloudflare Dashboard, select your domain. On the Overview page, scroll down to the right-hand API section — copy the Zone ID.

2. Create an API Token

Go to My Profile → API Tokens → Create Token.

Use the "Edit zone DNS" template as a starting point, then change the permission to:

Zone → Security Level → Edit

Restrict Zone Resources to the specific zone if you prefer. Click Create Token and copy the token — it is shown only once.

3. Enter values above and save

Paste the Zone ID and API Token into the fields below, then click . The panel will confirm the token is stored (it is never displayed again).

Auto Under Attack Mode — Brute Force

When multiple distinct IPs fail authentication simultaneously, automatically enables Cloudflare Under Attack Mode, then disables it once the storm calms.

Auto Under Attack Mode — L7 HTTP Flood

Tracks every HTTP request per IP. When several IPs exceed the per-minute threshold simultaneously (volumetric flood), UAM is triggered the same way as on brute force. Both conditions are checked independently when deciding to auto-disable.
